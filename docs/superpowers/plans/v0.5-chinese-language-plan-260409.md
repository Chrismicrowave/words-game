# v0.5 — Chinese Language Support Plan
**Branch:** `feature/chinese-language`  
**Date:** 2026-04-09  
**Status:** Planning

---

## Overview

Add Chinese language support with a letter-by-letter pinyin input mechanic:
- Player types individual pinyin letters (no tone marks)
- When a full syllable completes, the corresponding Chinese character "snaps in" to occupy that space
- Target display shows characters with optional pinyin-above hints (toggle)
- Word list panel shows characters only
- Language modes: English / Chinese / Mixed (extensible for future languages)

### Example — 你好 (nǐ hǎo)
```
Steps:  n  →  i  →  h  →  a  →  o
Display: [n] → [你] → [你h] → [你ha] → [你好]
```

## Stage 1 — Data Model

### 1.1 LanguageMode enum
```csharp
// Assets/-Scripts/Core/LanguageMode.cs
public enum LanguageMode { English, Chinese, Mixed }
```

### 1.2 ChinesePhase model
```csharp
// Assets/-Scripts/WordList/ChinesePhaseEntry.cs
[Serializable]
public class ChinesePhaseEntry
{
    public string character;   // e.g. "你"
    public string pinyin;      // e.g. "ni"  (no tone marks, lowercase)
}
```

### 1.3 Word list JSON schema extension
Existing format (English):
```json
{ "name": "My List", "words": ["hello", "world"] }
```

New format (Chinese):
```json
{
  "name": "HSK 1",
  "languageMode": "Chinese",
  "words": [],
  "chineseWords": [
    { "display": "你好", "entries": [{"character":"你","pinyin":"ni"},{"character":"好","pinyin":"hao"}] },
    { "display": "谢谢", "entries": [{"character":"谢","pinyin":"xie"},{"character":"谢","pinyin":"xie"}] }
  ]
}
```

Mixed format:
```json
{
  "languageMode": "Mixed",
  "mixedWords": [
    { "segments": [
        { "type": "english", "text": "hello" },
        { "type": "chinese", "entries": [{"character":"你","pinyin":"ni"},{"character":"好","pinyin":"hao"}] }
    ]}
  ]
}
```

### 1.4 FileWordListProvider update
- Read `languageMode` field, default to `English` if absent (backwards compatible)
- Parse `chineseWords` and `mixedWords` arrays
- Expose `LanguageMode LanguageMode { get; }`

### 1.5 IWordListProvider update
```csharp
LanguageMode LanguageMode { get; }
List<string> GetWords();                    // English: returns words; Chinese: returns display strings
List<ChineseWordEntry> GetChineseWords();   // null for English lists
```

## Stage 2 — WordEngine: Chinese Mode

### 2.1 SyllableBoundaryMap
For each Chinese phase, precompute:
- `string typeTarget` — concatenated pinyin, e.g. `"nihao"`
- `int[] boundaries` — cumulative letter counts where each character completes, e.g. `[2, 5]`
- `string[] characters` — the characters at each boundary, e.g. `["你", "好"]`

```csharp
public struct ChinesePhaseData
{
    public string typeTarget;        // "nihao"
    public int[] boundaries;         // [2, 5]
    public string[] characters;      // ["你", "好"]
    public ChinesePhaseEntry[] entries;
}
```

### 2.2 WordEngine changes
- Add `bool IsChinesePhase { get; private set; }`
- Add `ChinesePhaseData CurrentChineseData { get; private set; }`
- When loading a Chinese phase: build `ChinesePhaseData` from `ChineseWordEntry`
- `typeTarget` is passed to existing step-generation unchanged — no other engine changes needed
- Add `int CompletedCharacterCount` — returns how many boundaries the current typed position has passed

### 2.3 No tone validation needed
Input letters are plain a-z. Tone marks never appear in typeTarget. No changes to key validation.

## Stage 3 — Matched Display: CharacterCell Snap-In

### 3.1 CharacterCell prefab
```
CharacterCell (RectTransform, HorizontalLayoutGroup)
├── LetterLabel   (TMP — shows typed letters e.g. "ni", default font)
└── CharLabel     (TMP — shows Chinese character e.g. "你", Noto Sans SC, hidden until complete)
```
- When syllable incomplete: LetterLabel visible, CharLabel hidden
- When syllable complete: CharLabel visible, LetterLabel hidden
- Both labels sized so the cell width stays consistent (CharLabel uses same em-width as LetterLabel)

### 3.2 ChineseMatchedDisplay component
```csharp
// Assets/-Scripts/UI/ChineseMatchedDisplay.cs
public class ChineseMatchedDisplay : MonoBehaviour
{
    [SerializeField] private GameObject characterCellPrefab;
    [SerializeField] private Transform cellContainer;        // HorizontalLayoutGroup

    public void BuildCells(ChinesePhaseData data);          // spawns one cell per character
    public void UpdateProgress(int typedLetterCount);       // updates all cell states
    public void Clear();
}
```

Logic in `UpdateProgress(n)`:
- For each cell `i`: if `n >= boundaries[i]` → show CharLabel; else show LetterLabel with `typeTarget[prevBoundary..n]`

### 3.3 Layout — natural width
- `cellContainer`: HorizontalLayoutGroup, spacing=4
- Cells use natural content width — no fixed sizing needed
- Character snap-in will expand the cell (Chinese char is always wider than 1–2 latin letters); this jump is intentional and visually clear, like an IME

### 3.4 UIController integration
- In `UpdateTextDisplay()`: if `IsChinesePhase` → delegate to `ChineseMatchedDisplay`; else use existing TMP path
- `HandleStepProcessed` → call `UpdateProgress(wordEngine.MatchedLength)`

## Key Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| Input granularity | Letter-by-letter (5 steps for nihao) | Consistent with existing engine |
| Reveal trigger | Syllable boundary | Maps 1:1 to characters, natural feedback |
| Display approach | Separate ChineseMatchedDisplay component | Avoids patching TMP rich text; clean separation |
| Font | Noto Sans SC Dynamic TMP asset | Already in project, dynamic atlas = no pre-bake |
| Cell sizing | Natural width, jump allowed | Chinese chars are always wider than latin letters — jump is visually expected |
| Tone marks | Never — input and storage both tone-free | Simpler input, still correct for learning |
| Backwards compat | `languageMode` defaults to `English` if absent | Existing word lists unaffected |
